오디오/비디오 파일 삽입기
    오디오 파일 종류 : mp3, ogg, wav
    <audio> 태그 활용
        속성
        1. src : 오디오 파일의 위치
        2. controls : 재생 제어기 표시
        3. loop : 반복 재생 
        4. autoplay : 문서가 출력됨과 동시에 자동 재생
                        (저작권 문제로 크롬에서는 자동 재생 안됨)
        5. muted : 음소거 상태로 시작
        6. preload : 재생 전에 미리 다운로드 

    보조태그 : <source> 재생안될때 확인하는 태그. 태그안의 오디오 파일을 순서대로 검사하면서
    지원하는 형식을 만나면, 검사를 멈추고 그 파일 재생함.
    예시> 
        <audio controls>
            <source src="bear.mp3" type="audio/mpeg">
            <source src="bear.ogg" type="audio/ogg">
            <source src="bear.wav" type="audio/wav">
            오디오 파일을 재생할 수 없습니다.
        </audio>
        1. bear.mp3를 재생.
        2. 만일 브라우저에서 mp3를 재생할 수 없으면, bear.ogg를 재생.
        3. 만일 브라우저에서 ogg를 재생할 수 없으면, bear.wav를 재생.
        4. 만일 모두 재생할 수 없으면, 마지막 문장을 출력

시맨틱 태그(semantic tag)
    페이지 영역에 의미를 부여하는 태그.
    css가 적용되지 않을 경우, 특별한 모양이나 형식을 보이지 않음
    페이지 완성 또는 작업 중, 유지보수 등의 작성 시 해당 영역을 쉽고 빠르게 찾아서 처리하는데 도움을 줌
    
    -태그 종류
    1)header : 머릿글
    2)nav  : 메뉴
    3)section : 본문영역
    4)article : 세부영역
    4)arsticle : 부가정보, 광고
    4)footer : 바닥글

+)theader나 tfoot처럼 테이블에서 사용되는 시멘틱 태그는 강제성 있지만, 
일반적인 경우, 강제성은 없음
편하려고 나눠놓은 느낌

<form> 태그 : 사용자의 입력을 다른 페이지나 서버 프로그램에 전달하기 위한 태그
			일종의 컨테이너 태그
			전달해야 할 정보를 묶어서 전달
	주요 속성
		1. action : 입력 값을 받을 대상(프로그램)을 지정 
(대상 jsp, asp, php, 프로그램) 
		2. method : 전달 방식을 지정. get/post
			     기본값은 get.


전달 방식의 차이 
GET: 주소창에 전송 데이터가 노출되는 방식
		보안상으로 문제가 없는 짧은 데이터 전성에 적합


POST: 주소창에 데이터가 노출되지 않는 방식. 
	길고, 보안상으로 문제가 있는 데이터 전송에 적합 /로그인은 post방식 취함

	
+form은 정보를 직접 받지 못함 -> input이랑 같이 쓰이는 경우가 많음
