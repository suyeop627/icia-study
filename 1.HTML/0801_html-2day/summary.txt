서식 처리용 태그(요소) - 프레이징 컨텐츠용
    글자의 모양을 바꾸는 태그
    1. 글자 태그
        1) 제목 태그 (h1~h6) : 제목 요소를 작성하는 태그(블록태그)
        2) 본문 태그 (p) : 본문 요소를 작성하는 태그(블록태그)
                           단축 태그로 줄바꿈 기능으로만 사용 가능.
    2. 서식 태그
        <b>, <i>, <strong>, <em>, <mark>, <sub>, <sup>, <ins>, <del>, <small> 등
    

참고) 자원(resource)
URL(uniform resource locator) : 인터넷에서 사용하는 주소
    1. 텍스트
    2. 이미지 : jpg, png, gif 등
    3. 음성 : wav, mp3 등
    4. 영상 : avi, mp4 등
    5. 객체(Object) : flash, 파일 등

링크 태그(Anchor, 닻) - <a> 
    다른 페이지와 연결하기 위한 요소를 만드는 태그.
    필수 속성 - href : 이동할 페이지를 지정하는 속성. 
                    해당 페이지ㅡ이 url을 작성. 
                    url의 일반적인 형식 - https://www.naver.com  <- 프로토콜포함해서 전체 주소 입력
                    https - 프로토콜(규약, protocol) : 전송 규약. 주고 받는 양 측의 정보의 차이가 없도록 도와줌
    기타속성 - target : 이동할 페이지를 어디에 보여질 지 지정.
                속성값 : _self, _blank

    북마크 기능(book mark) - 책갈피 기능
        같은 페이지의 보이는 부분을 변경.
        id 속성을 활용

    이미지 추가 태그 : <img>
        이미지 파일을 문서에 추가하기 위한 태그. 속성을 활용
        src : 삽입할 이미지의 경로(위치)와 파일명 설정.
        alt : 이미지가 출력되지 않을 경우 대신 출력할 문장을 작성.
        title : 이미지 위에 마우스가 오버되었을 경우 출력할 문장을 작성
        width : 이미지 출력 시 너비를 지정하는 속성.
        height : 이미지 출력시 높이를 지정하는 속성.(잘 지정하지 않음, 너비만 지정할 때가 많음, 높이 미지정시, width와 비율에 따라 정해짐)
    



요소의 속성(attribute)은 시작 태그에만 작성함.


픽셀(pixer) 
    화소. 화면을 구성하는 최소이며, 기본 단위.(px)
    위치(x,y) 정보와 색상(RGB)정보로 구성.->bitmap.(bmp)
    화면의 해상도는 픽셀의 개수로 나타난다.

참고) 편집화면과 실화면의 차이
    -편집화면에서 줄바꿈은 실화면에서 처리되지 않고, 다만 한칸의 빈 공백이 들어간다.
    -편집화면에서 여러 칸의 빈 공백도 한칸의 빈 공백으로 처리된다.


SPA(singl page application)
    html페이지는 하나만 만들어놓고 그 안에서 계속 수정되는거
    페이지를 이동하지 않음(JS활용)
